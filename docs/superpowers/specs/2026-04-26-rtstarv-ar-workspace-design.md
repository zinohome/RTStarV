# RTStarV AR 工作空间 — 设计文档

## 概述

RTStarV 是一个跨平台 AR 工作空间应用，让魅族 StarV View AR 眼镜成为真正的生产力工具。替代魅族 MYVU App，提供高质量的虚拟多屏幕办公体验。

**目标平台：** Android（小米 14 Pro）/ macOS / Windows  
**技术方案：** Unity 跨平台  
**核心理念：** 眼镜是纯显示器 + IMU 传感器，所有计算在 host 设备完成

---

## 硬件约束：StarV View

- 索尼 OLED，单眼 1920×1080，120Hz，峰值 700nits
- BirdBath 光学，43.5° FOV，51 PPD
- 边缘清晰度好 — 软件必须原生 1080p 直出，不做缩放和畸变处理
- USB-C（DP Alt Mode 视频 + USB HID 数据）
- TDK ICM-42688-P 6 轴 IMU 传感器，3DoF
- Lontium LT7911UXC DP 转 MIPI 芯片
- 重量 74g，近视 0-600° 无级调节
- **无操作系统，无计算能力，无法运行应用**

## 虚拟屏幕布局

### 三种布局模式

**1 屏模式（居中）：**
- 单个虚拟屏幕，正前方 0°

**3 屏模式（横向并排）：**
- 三个虚拟屏幕，分别位于 左 -35° / 正前 0° / 右 +35°

**6 屏模式（两排三列）：**
- 上排三个，下排三个
- 横向间距 35°，纵向间距 25°

### 关键参数

- 屏幕间距角度：约 35°（FOV 43.5°，留重叠区域自然过渡）
- 上下排间距角度：约 25°（轻微抬头/低头切换）
- 每个虚拟屏幕 = 操作系统的一个虚拟显示器，分辨率 1920×1080

## 空间交互

### 悬停模式（默认且唯一模式 — 电脑端）

虚拟屏幕固定在空间中，头部转动只改变视角。

### 手机端支持两种模式

- 跟头模式：屏幕跟随头部转动
- 悬停模式：屏幕固定在空间中

### 一键居中（核心功能）

- 电脑端快捷键 `Ctrl+Shift+Space`
- 手机端底部居中按钮
- 整个虚拟空间瞬间重新定位，中间屏幕对准当前头部朝向
- 平滑过渡动画，约 200ms

### 电脑端快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+Space` | 一键居中 |
| `Ctrl+Shift+1/2/3` | 切换 1/3/6 屏模式 |
| `Ctrl+Shift+←/→` | 焦点切换到左/右屏幕 |
| `Ctrl+Shift+↑/↓` | 焦点切换到上/下屏幕（6屏模式） |

### 布局切换行为

- 切换时先自动居中，然后新屏幕以动画滑入/滑出（约 300ms）
- 增加屏幕：系统新增虚拟显示器，新屏幕滑入
- 减少屏幕：多余虚拟显示器移除，窗口自动合并到剩余显示器（OS 默认行为）
- 全程可见，不黑屏

## 手机端交互

### 屏幕布局

```
┌──────────────┐
│ 跟头 | 悬停   │  ← 顶部按钮栏
├──────────────┤
│              │
│              │
│   触摸区域    │  ← 主操作区
│              │
│              │
├──────────────┤
│   居中按钮    │  ← 底部按钮
└──────────────┘
```

### 触摸操作

- 单指滑动 → 移动光标
- 单指点击 → 左键点击
- 长按 → 右键点击
- 双指滑动 → 滚动

### 居中按钮

- 点击 → 居中屏幕（整个虚拟空间回到视线正前方）
- 长按 → 居中光标（光标回到屏幕中央）

### 输入法

- 光标进入文字输入区域时，系统输入法正常弹出在最底部
- 居中按钮被输入法顶上去
- 触摸区域不动，不做特殊处理，遵循 Android 标准行为

## IMU 驱动

### 数据流

```
StarV View (ICM-42688-P) @ ~400Hz
    → USB HID
    → C++ 原生插件 (hidapi)
    → 姿态解算（互补滤波）
    → 低通滤波去抖（截止 10-15Hz）
    → Unity Camera yaw/pitch 旋转
    → 渲染输出 → DP → 眼镜
```

### 逆向工程

- 用 Wireshark + USBPcap 抓取 MYVU App 与眼镜的 USB 通信
- 参考 Xreal/Rokid 已破解协议（架构类似）
- 确认 VID/PID、IMU endpoint、激活指令字节序列

### 性能要求

- 全链路延迟 < 20ms
- IMU 处理 < 2-3ms
- 120Hz 渲染输出

## 虚拟显示器与画面捕获

### 电脑端

**创建虚拟显示器：**
- Windows: IddSampleDriver（微软官方虚拟显示器框架）
- macOS: CGVirtualDisplay API（macOS 13+）
- 每个虚拟显示器 1920×1080，系统视为真实显示器

**画面捕获：**
- Windows: DXGI Desktop Duplication API
- macOS: ScreenCaptureKit
- 捕获画面直接作为 GPU 纹理，避免 CPU 拷贝

### 手机端

- Android MediaProjection API 捕获屏幕内容
- 启用 Android 桌面模式，横屏桌面化
- 单虚拟屏幕

### 渲染优化

- 只渲染可见屏幕（视野外暂停捕获）
- 纹理 GPU 直传
- 捕获 60fps，渲染输出 120fps

## 子项目拆分与开发顺序

### 子项目 1：IMU 驱动层（最优先 — 项目验证点）

- 逆向 StarV View USB IMU 协议
- C++ hidapi 原生插件
- 姿态解算 + 滤波
- **此项不通则项目不可行**

### 子项目 2：Unity 3D 空间引擎

- 虚拟屏幕 3D 定位和布局管理（1/3/6 屏）
- IMU → 摄像机旋转映射
- 悬停模式、一键居中、快捷键体系
- 1080p @ 120Hz DP 输出

### 子项目 3：电脑端虚拟显示器 + 画面捕获

- Windows: IddSampleDriver + DXGI
- macOS: CGVirtualDisplay + ScreenCaptureKit
- 捕获画面 → Unity 纹理映射

### 子项目 4：Android 手机端

- 触摸板控制器
- 桌面模式渲染（MediaProjection）
- 居中按钮、输入法联动

### 依赖关系与并行

```
子项目 1 (IMU 驱动) ──验证通过──→ 子项目 2 (3D空间引擎)
                                        │
                                ┌───────┴───────┐
                                ▼               ▼
                        子项目 3 (电脑端)    子项目 4 (手机端)
                        可并行开发          可并行开发
```

### MVP

最小可用版本 = 子项目 1 + 2 + 3（Windows 版）：
- StarV View 连接 Windows 电脑
- 1 个虚拟屏幕，悬停模式，一键居中
- 验证核心体验

扩展路径：3/6 屏模式 → macOS 版 → Android 版
